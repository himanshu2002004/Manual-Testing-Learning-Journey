# Day 3 - Software Testing Life Cycle (STLC)

---

# What is STLC?

Software Testing Life Cycle (STLC) is a systematic process followed by the testing team to ensure that software is thoroughly tested before it is released to users.

In simple words, STLC is a step-by-step process that guides testers from understanding the requirements to completing the testing process.

---

# Definition

Software Testing Life Cycle (STLC) is a sequence of activities performed during the software testing process to ensure that the software meets the required quality standards.

---

# Why is STLC Important?

### 1. Improves Software Quality

### 2. Detects Defects Early

### 3. Provides Structured Testing

### 4. Improves Test Coverage

### 5. Ensures Customer Satisfaction

### 6. Reduces Project Risks

---

# Objectives of STLC

- Verify application functionality.
- Identify defects.
- Ensure all requirements are tested.
- Improve software quality.
- Deliver a bug-free application.
- Increase customer satisfaction.

---

# Phases of STLC

1. Requirement Analysis
2. Test Planning
3. Test Case Development
4. Test Environment Setup
5. Test Execution
6. Test Closure

---

# STLC Flow

Requirement Analysis
     ↓
Test Planning
     ↓
Test Case Development
     ↓
Test Environment Setup
     ↓
Test Execution
     ↓
Test Closure

---

# Phase 1: Requirement Analysis

Requirement Analysis is the first phase of STLC.

In this phase, testers study the Software Requirement Specification (SRS) document and understand the application's functionality.

The objective is to identify what needs to be tested.

---

## Activities Performed

- Study SRS document.
- Understand business requirements.
- Identify testable requirements.
- Identify testing scope.
- Clarify doubts with the Business Analyst or Client.

---

## Deliverables

- Requirement Analysis Document
- Requirement Traceability Information
- List of Testable Requirements

---

# Phase 2: Test Planning

Test Planning is the phase where the complete testing strategy is prepared.

The Test Lead or Test Manager creates a Test Plan.

---

## Activities Performed

- Define testing scope.
- Select testing types.
- Estimate testing effort.
- Allocate resources.
- Decide testing schedule.
- Identify risks.

---

## Deliverables

- Test Plan Document
- Resource Plan
- Test Schedule

---

# Phase 3: Test Case Development

Test cases are created based on requirements.

These test cases are later executed during testing.

---

## Activities Performed

- Write Test Cases.
- Review Test Cases.
- Prepare Test Data.
- Update Requirement Traceability Matrix (RTM).

---

## Deliverables

- Test Cases
- Test Data
- RTM
  
---

# Phase 4: Test Environment Setup

A testing environment is prepared before executing test cases.

The testing environment should be similar to the production environment.

---

## Activities Performed

- Install application.
- Configure server.
- Configure database.
- Install testing tools.
- Verify environment readiness.

---

## Deliverables

- Test Environment
- Environment Readiness Report

---

# Phase 5: Test Execution

Testers execute all prepared test cases.

If defects are found, they are reported to developers.

---

## Activities Performed

- Execute Test Cases.
- Record Results.
- Report Bugs.
- Retest Fixed Bugs.
- Perform Regression Testing.

---

## Deliverables

- Test Execution Report
- Defect Report
- Test Status Report

---

# Phase 6: Test Closure

Test Closure is the final phase of STLC.

The testing team evaluates the testing process and prepares a final report.

---

## Activities Performed

- Verify completion of testing.
- Prepare Test Summary Report.
- Analyze defect statistics.
- Document lessons learned.

---

## Deliverables

- Test Closure Report
- Test Metrics
- Final Sign-Off
  
---

# Entry Criteria and Exit Criteria in STLC

## What is Entry Criteria?

Entry Criteria are the conditions that must be fulfilled before starting a testing phase.

In simple words, Entry Criteria answer the question:

**"What should be ready before we begin this phase?"**

If the entry criteria are not met, the testing phase should not start.

---

## What is Exit Criteria?

Exit Criteria are the conditions that must be completed before ending a testing phase.

In simple words, Exit Criteria answer the question:

**"How do we know this phase is completed successfully?"**

If the exit criteria are not met, the testing phase cannot be considered complete.

# Real-Time Example: Testing an E-Commerce Website

### Requirement Analysis

Study requirements for:

- User Registration
- Login
- Search Products
- Add to Cart
- Payment
- Order History

### Test Planning

Plan:

- Functional Testing
- Regression Testing
- Smoke Testing

Estimate:

- 4 Test Engineers
- 20 Days

### Test Case Development

Prepare test cases for:

- Login
- Search
- Cart
- Checkout
- Payment

### Test Environment Setup

Configure:

- QA Server
- Database
- Chrome Browser
- Test User Accounts

### Test Execution

Execute all test cases.

Report defects if any functionality fails.

### Test Closure

Prepare:

- Test Summary Report
- Defect Summary
- Lessons Learned

# Difference Between SDLC and STLC

|             SDLC                        |             STLC                 |
|-----------------------------------------|----------------------------------|
| Software Development Life Cycle         | Software Testing Life Cycle      |
| Focuses on software development         | Focuses on software testing      |
| Performed by Developers, Architects, BA | Performed by Test Engineers      |
| Starts with Requirement Gathering       | Starts with Requirement Analysis |
| Ends with Maintenance                   | Ends with Test Closure           |

---
