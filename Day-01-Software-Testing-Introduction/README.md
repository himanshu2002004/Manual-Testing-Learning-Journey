# Day 1 - Introduction to Software Testing

## What is Software Testing?
Software Testing is the process of verifying and validating a software application to ensure it works according to business requirements and is free from defects.

---

## Why is Software Testing Important?

1. Improves Software Quality
Testing helps identify defects and ensures the application works correctly.

2. Reduces Production Issues
Finding bugs during testing is less expensive than fixing them after release.

3. Enhances User Experience
A well-tested application provides a smooth and reliable experience for users.

4. Ensures Business Requirements are Met
Testing verifies that the application satisfies customer requirements.

5. Improves Security and Reliability
Testing helps identify vulnerabilities and system failures.

---

## Objectives of Software Testing

✔ Find defects before the application reaches users.

✔ Verify that all requirements are implemented correctly.

✔ Validate that the application behaves as expected.

✔ Improve software quality and reliability.

✔ Increase customer satisfaction.

---

## Real-Time Example

Application: Login Page

Consider an application that has a Login page where users enter their username and password to access their account.

As a Test Engineer, I would perform the following checks:

1. Open the Login page and ensure that the Username field, Password field, and Login button are visible.
2. Enter the correct username and password and verify that I can log in successfully.
3. Enter an incorrect password and check whether an appropriate error message is displayed.
4. Leave the fields empty and verify that the application asks me to enter the required information.
5. Enter only the username and check whether the application requests the password.
6. Click the Login button and make sure the application responds properly without errors.

This simple example helps in understanding how testing is used to verify that an application works correctly and provides a good experience to users.
