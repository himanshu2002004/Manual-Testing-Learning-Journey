# Amount Transfer Test Cases

## Overview

This repository contains manual test cases for the **Amount Transfer** functionality of a banking application. The test cases are designed based on the functional and business requirements to ensure complete validation of the fund transfer process.

## Requirements

The Amount Transfer module should allow users to:

- Transfer funds between valid accounts.
- Enter a valid **From Account Number**.
- Enter a valid **To Account Number**.
- Enter a valid transfer amount.
- Perform the transaction using the **Transfer** button.
- Cancel the transaction using the **Cancel** button.

## Test Design Approach

The test cases were designed using a **Requirement-Based Testing** approach. Requirements were analyzed before creating the test cases to ensure complete functional coverage. These test cases were **not created using exploratory testing**.

## Test Design Techniques Used

- Positive Testing
- Negative Testing
- Boundary Value Analysis (BVA)
- Equivalence Partitioning (EP)
- Validation Testing
- Requirement-Based Testing

## Functionalities Covered

- From Account Number Validation
- To Account Number Validation
- Amount Validation
- Successful Fund Transfer
- Invalid Account Number Validation
- Same Account Validation
- Transfer Button Functionality
- Cancel Button Functionality


## Objective

To verify that the Amount Transfer module functions correctly according to the specified requirements by validating all business rules, input validations, and transaction workflows.
